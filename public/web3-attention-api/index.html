新鮮勢
不是一直重做網站

是在把價值往核心收斂
Demo deck 草案｜給非技術聽眾

問題不是資訊不夠

Web3 每天都有新聞、公告、社群討論、鏈上事件、監管進展。

真正麻煩的是：今天到底哪幾件事值得注意？
資訊很多
來源越來越廣，真假與重要性混在一起。
注意力有限
人一天只能處理少數事件。
需要排序
先知道什麼值得看，再決定要不要深入。

第一版：手刻 HTML

一開始，新鮮勢是一個直接手刻出來的前端頁面。

目標
先把事件卡片、摘要、來源、社群討論放到讀者眼前。
價值
證明這套閱讀方式可行。
問題
頁面、資料與互動綁太緊，後續很難維護。

第二版：換成 Astro

Astro 讓新鮮勢從手刻頁面，變成可維護的網站。

網站變穩

首頁、permalink、tag、archive 可以系統化產生。

資料還是核心

`docs/feed.json` 仍然是事件資料來源。Astro 只是官方閱讀介面。

第三版：Web3 Attention API

現在要再往下一層走。前端可以換，但背後的判斷要穩定。

Event
發生了什麼事。
Source
可信依據從哪裡來。
Attention
為什麼值得現在被看見。
排序 篩選 來源權重 Agent 可讀

網站變成官方樣板

未來新鮮勢網站不是唯一產品，而是 Web3 Attention API 的 reference client。

層級負責的事誰會用
API事件、來源、分數、排序理由Agent、企業、研究團隊
Astro 網站把同一套判斷示範給人看一般讀者、潛在客戶
客戶前端依自己的品牌與工作流呈現付費客戶

為什麼客戶可能買單

客戶不一定想買一個新聞網站。他們可能想買一套能接進自己工作流的判斷。

投資團隊
每天先看最重要的 20 件事。
交易所
追蹤政策、資安與市場事件。
AI Agent
定時讀取 API，自動產出 briefing。

這幾天到底在忙什麼

表面上是在抽換架構，其實是在把產品從「畫面」推進到「資料判斷」。

手刻 HTML
先證明這個閱讀方式有人看得懂。
Astro
把網站變成穩定的官方介面。
Attention API
讓同一套判斷可以被機器與客戶使用。

最小可行成果

`GET /v1/attention/daily.json`

這個 endpoint 回傳今天最值得注意的 10 到 30 件 Web3 事件。

  • 每件事有摘要。
  • 每件事有來源。
  • 每件事有注意力分數。
  • 每件事有排序理由。

一句話結論

新鮮勢賣的不是頁面，而是 Web3 世界裡「什麼值得注意」的判斷。

網站讓人看懂這套判斷。API 讓客戶與 Agent 把這套判斷放進自己的工具。
